## What is a Project?

A project is the primary organizational unit in the platform. Everything you create — articles, outlines, images, knowledge sources, social media posts — lives inside a project. Think of a project as a container for a single website, brand, or content strategy.

Most users have one project per website they manage. Agencies and teams often have one project per client.

## Creating Your First Project

When you first sign up, a default project is created for you automatically. You can rename it and configure it to match your actual use case.

## Creating Additional Projects

1. Click the **project name** in the top-left of the sidebar to open the project switcher.
2. Click **New Project** at the bottom of the switcher dropdown.
3. Enter a project name. Choose something descriptive — "Acme Corp Blog" is more useful than "Project 1".
4. Optionally enter a description and target website URL.
5. Click **Create Project**.

The new project opens immediately and you are set as its owner.

> **Tip:** The website URL you enter during creation is used as a default when configuring WordPress publishing and Google Search Console integrations. Set it accurately to save time during integration setup.

## Naming Projects

Project names appear throughout the interface — in the sidebar, in the project switcher, in team invitations, and in exported reports. Use names that are immediately recognizable to all team members who will work in the project.

Good naming patterns:
- Client name: "Riverside Dental"
- Client name + type: "Riverside Dental — Blog"
- Brand + market: "Acme Corp — US English"

Avoid generic names like "New Project" or "Test" — these become confusing once you have several projects.

## Switching Between Projects

Use the project switcher in the top-left of the sidebar to move between projects. Clicking a project name takes you to that project's dashboard immediately.

The platform preserves your last-visited section within each project. If you were editing an article in Project A, then switched to Project B, switching back to Project A will return you to the articles section.

> **Tip:** Keyboard shortcut: press `P` from the dashboard to open the project switcher without using the mouse.

## What Lives Inside a Project

Each project contains:

| Section | Contents |
|---------|---------|
| Articles | All published, draft, and scheduled articles |
| Outlines | Content outlines and brief structures |
| Images | AI-generated and uploaded image library |
| Knowledge Vault | Uploaded reference documents for AI-powered content generation |
| Social Media | Scheduled and published social posts |
| Analytics | SEO performance, AEO scores, content decay alerts |
| Settings | Integrations, brand voice, team members |

Nothing is shared between projects automatically. If you want the same content or knowledge source in two projects, you add or upload it to each project separately.

## Project Settings

Access project-level settings by clicking the gear icon next to the project name in the sidebar, or navigating to **Settings** within the project.

From project settings you can:
- Rename the project or update the description
- Set a target keyword or primary topic for the project
- Configure brand voice settings for AI generation
- Manage team member access and roles
- Connect integrations (WordPress, Google Search Console, social accounts)
- Archive or delete the project

## Archiving and Deleting Projects

**Archive** — Hides the project from the active project list without permanently deleting it. All content is preserved and the project can be restored later. Useful for pausing a client engagement.

**Delete** — Permanently removes the project and all its content including articles, images, outlines, and knowledge sources. This action cannot be undone. You will be asked to type the project name to confirm before deletion proceeds.

> **Tip:** If a client engagement ends, archive rather than delete the project. Archived projects preserve the content history, which can be valuable if the engagement resumes or if you need to reference past work.
