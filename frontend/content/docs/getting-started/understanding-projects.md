## What is a Project?

A project is the primary organizational container in A-Stats. Think of it as a workspace scoped to a single website, publication, or client. Everything you create in A-Stats — outlines, articles, images, Knowledge Vault documents, social posts, GSC connections — belongs to a project.

Projects keep your work clean and separate:

- Each project has its own **brand voice settings** so AI-generated content matches the tone and style of that specific site
- Each project has its own **team members** with individual roles and permissions
- Each project has its own **integrations** (WordPress, Google Search Console, social accounts)
- Analytics, content decay alerts, and usage data are all scoped to the project

If you manage one website, you will probably have one project. If you manage content for multiple clients or run multiple publications, you will have one project per client or site.

> **Tip:** Projects are free to create — there is no limit on the number of projects in your account. Your subscription limits apply at the account level, not per project.

---

## Creating a Project

1. From any screen in A-Stats, click the **project switcher** in the top-left of the sidebar (it shows your current project name).
2. Click **New Project** at the bottom of the dropdown.
3. Enter a **project name**. Use something descriptive — the name of the website, the client name, or the content vertical.
4. Optionally set a **description**. This is only visible to team members and is useful for agencies managing multiple client projects.
5. Click **Create**. You will be taken into the new project workspace immediately.

After creation, visit **Project Settings** to configure brand voice, integrations, and team access.

---

## Switching Between Projects

The project switcher is in the top-left corner of the sidebar and shows your currently active project. Click it to open a dropdown listing all projects you have access to.

- Projects where you are an **owner or admin** appear at the top
- Projects where you are an **editor or viewer** appear below
- Click any project name to switch to it instantly

All sidebar navigation (Outlines, Articles, Knowledge, Social, Analytics) reflects the currently selected project.

## Team Roles

Projects support four roles. Each role is additive — editor permissions include everything viewer has, admin includes everything editor has, and so on.

### Viewer
Can read all content in the project — outlines, articles, knowledge sources, and analytics. Cannot create, edit, or delete anything. Suitable for clients or stakeholders who need visibility without write access.

### Editor
Can create and edit outlines and articles, use the AI generation tools, upload to the Knowledge Vault, and generate images. Cannot manage team members, change project settings, or delete the project. This is the right role for writers and content creators.

### Admin
Can do everything an editor can, plus: invite and remove team members, change project settings (brand voice, integrations), manage social account connections, and view billing-related project information. Admins cannot transfer project ownership or delete the project.

### Owner
Full control. The owner can do everything admins can, plus delete the project and transfer ownership to another member. Each project has exactly one owner at a time.

> **Tip:** When adding a contractor or freelance writer, use the **Editor** role. When adding a client who just wants to review content, use **Viewer**. Reserve **Admin** for internal team leads who manage the project configuration.

## Inviting Team Members

1. Go to **Project Settings > Members**.
2. Click **Invite Member**.
3. Enter the email address of the person you want to invite.
4. Select their role from the dropdown (Viewer, Editor, or Admin).
5. Click **Send Invite**.

The invited person will receive an email with a link to accept the invitation. If they do not have an A-Stats account, the link will take them through account creation first.

**Managing existing members:**

- To change a member's role, click the role badge next to their name and select a new role
- To remove a member, click the three-dot menu next to their name and select **Remove**
- You can view all pending invitations and resend or cancel them from the same Members page

### Leaving a Project

If you are an editor, admin, or viewer (not the owner), you can leave a project at any time:

1. Go to **Project Settings > Members**
2. Scroll to your own entry at the bottom
3. Click **Leave Project**

The owner cannot leave a project — they must first transfer ownership to another member.

### Transferring Ownership

1. Go to **Project Settings > Members**
2. Find the member you want to make the new owner
3. Click the three-dot menu next to their name
4. Select **Transfer Ownership**
5. Confirm the transfer

After transfer, your role becomes Admin. Only the new owner can transfer ownership again.

## Brand Voice Settings

Brand voice tells the AI how to write for this project. It is configured at the project level so all editors producing content for the same project get consistent output.

Go to **Project Settings > Brand Voice** to configure:

- **Writing style**: Informative, Conversational, Technical, or Persuasive
- **Voice**: First-person singular, first-person plural, or third-person neutral
- **List usage**: How heavily to use bullet and numbered lists
- **Custom instructions**: Free-text instructions applied to every AI generation in this project (e.g., "Always use Oxford commas" or "Avoid jargon — write for a non-technical audience")

These settings are the defaults for every outline and article created in the project. Individual outlines can override them if needed.

> **Tip:** For agencies managing client projects, the Custom Instructions field is where you put client-specific rules: preferred terminology, topics to avoid, style guide requirements, and competitor names not to mention.

## Project Integrations

Each project can connect independently to external services. Configure integrations under **Project Settings > Integrations**.

### Google Search Console
Connect your GSC property to pull search performance data for the articles in this project. Each project connects to one GSC property. If you manage multiple sites, each gets its own project and its own GSC connection.

### WordPress
Connect a WordPress site using an Application Password (not your main account password). Once connected, you can publish articles directly from the A-Stats editor to your WordPress site as drafts or published posts.

### Social Accounts
Connect LinkedIn, X (Twitter), or Facebook accounts to enable the social scheduler for this project. Each project maintains its own social connections, so agency clients with separate social presences can be managed cleanly.

---

## Deleting a Project

Deleting a project permanently removes all its content — outlines, articles, knowledge documents, images, and analytics data. This action cannot be undone.

Only the project owner can delete a project.

1. Go to **Project Settings > Danger Zone**
2. Click **Delete Project**
3. Type the project name to confirm
4. Click **Confirm Delete**

> **Tip:** If you want to archive a project rather than delete it, consider removing team members and simply leaving it inactive. All content remains accessible to the owner.
